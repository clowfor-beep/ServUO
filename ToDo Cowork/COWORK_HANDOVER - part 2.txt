AI CHARACTER TYPE REFERENCE (CRITICAL)

TYPE A — SimPlayers (persistent, 144 total)
  Class: SimPlayer : BaseFBCombatNPC

  On death: cooldown timer, re-activates at HomeLocation
  Max active: ~30 simultaneously
  Examples: All 12 SimGuild named members

  Class: PoolPK : BaseFBCombatNPC
  Managed by: FBPKSpawner
  On death: true respawn after timer (new instance)
  Examples: "a brigand", "a ruffian", dungeon grunts

TYPE C — Encounter PKs (triggered, temporary)
  Class: NovicePlayerKiller : BaseFBCombatNPC (existing)
  Managed by: GraveyardPKEncounter / FBEncounterSystem
  On death: despawn (5-min auto-delete)
  Examples: Britain graveyard PK, dungeon ambushes

DO NOT confuse these. Different lifecycles, different managers.


================================================================
EQUIPMENT LAYER RULE (DO NOT SKIP)
================================================================

Evidence: LayerConflict.log shows production bugs from 18-05-2026
  SampireNewbie: NecromancerSpellbook + Katana = layer conflict
  PaladinNewbie: BookOfChivalry + Broadsword = layer conflict

THE RULE:
  Weapons:        SetWearable(new Katana())         explicit layer
  Shields:        SetWearable(new HeaterShield())   explicit layer
  Armour:         SetWearable(new PlateChest())     explicit layer
  Spellbooks:     AddToBackpack(new Spellbook())    BACKPACK ONLY
  NecroSpellbook: AddToBackpack(new NecroBook())    BACKPACK ONLY
  BookOfChivalry: AddToBackpack(new BookOfChiv())   BACKPACK ONLY

ServUO NPC AI can cast from backpack — this is safe.
Never use FirstValid layer for books. Always AddToBackpack.
This affects: NecroMage, Sampire, Paladin, Pure Mage templates
