# ── 2. COOLDOWN HUD SYSTEM ───────────────────────────────────
# File: Scripts/Custom/CooldownSystem.cs
# Status: COMPLETE and compiling

How it works:
  CooldownSystem.Start(mobile, skillName, seconds) — call from anywhere
  CooldownGump updates every 0.5s
  Shows skill name + X.X countdown + progress bar
  Colors: green (>3s) → orange (1-3s) → red (<1s)
  Gump position: x=250, y=600 (configurable in CooldownConfig)
  Draggable, non-closable, auto-removes when timer expires

Hook architecture:
  Skills.cs (Server/Skills.cs) has a static delegate:
    public static Action<Mobile, string, double> OnSkillUsed;
  CooldownSystem.Initialize() hooks into this delegate at startup
  This fires for ALL player-activated skills with cooldown >= 1.0s

Known limitation:
  Begging and ItemID set NextSkillTime internally, bypassing the hook
  PENDING FIX: add CooldownSystem.Start directly inside those skill files
